Полный список файлов аукционного модуля «Найдук»
📁 Страницы пользовательского интерфейса
1. /auctions/create.php
Назначение: Страница создания нового аукциона (прямого или обратного).

Функционал:

Выбор типа аукциона (прямой/обратный).

Заполнение заголовка, описания, загрузка фото (до 5), выбор категории и города.

Для прямого аукциона: стартовая цена, резервная цена, шаг (авто/фикс), длительность, скрытые ставки, авто‑предложение второму, 3D‑эффект, цена мгновенной покупки.

Для обратного: желаемая цена покупки, срок приёма предложений.

Телефон продавца, видимость номера.

Расчёт платы за размещение (30–100 ₽) и кнопка «Оплатить и опубликовать».

Лимит на количество черновиков (не более 2).

Что можно менять: Набор полей, логику расчёта платы, стили, количество фото.

2. /auctions/view.php
Назначение: Страница просмотра одного аукциона (прямого).

Функционал:

Отображение фото с 3D‑эффектом (если включён).

Таймер обратного отсчёта с обновлением каждую секунду.

Текущая ставка (скрытая/открытая).

Поле ввода ставки с быстрыми кнопками (+500, +1000, +5000).

Индикатор оставшихся ставок (бесплатные 3 + платные из глобального баланса).

Кнопка «Сделать ставку» (с проверками).

Кнопка «Купить сейчас» (если продавец установил цену).

История ставок с анонимными ID и цветами.

Модальное окно согласия с правилами.

Подключение к SSE для мгновенных обновлений.

Для продавца: отображение предупреждения, что он не может ставить.

Что можно менять: Лимиты ставок, количество отображаемых записей истории, стиль анимации, поведение таймера.

3. /auctions/index.php
Назначение: Каталог активных аукционов.

Функционал:

Фильтрация: тип (прямой/обратный), категория, город, цена (от/до).

Сортировка: скоро завершатся, новые, по цене (возр./убыв.).

Пагинация (20 лотов на страницу).

Карточки: фото, название, текущая ставка, количество ставок, таймер.

Ссылка на создание аукциона.

Что можно менять: Количество элементов на странице, набор фильтров, внешний вид карточек.

4. /auctions/my.php
Назначение: Личный кабинет участника аукционов.

Функционал:

Вкладки: «Мои аукционы» (созданные), «Мои ставки» (участие), «Ожидают подтверждения» (покупки/продажи).

Для каждого аукциона: кнопки «Перейти», «Перевыставить бесплатно», «Продать победителю», «Удалить».

Для сделок: кнопки «Подтвердить получение» / «Подтвердить отправку».

Отображение непогашенной комиссии.

Что можно менять: Визуальное оформление вкладок, тексты кнопок, логику отображения.

5. /auctions/accept_offer.php
Назначение: Страница принятия/отказа от предложения второму участнику (после неоплаты победителя).

Функционал:

Отображает информацию о предложении (лот, цена, срок действия).

Кнопки «Принять» и «Отказаться».

При принятии: создаётся сделка, обновляется аукцион, отправляются уведомления.

При отказе: предложение помечается как отклонённое.

Что можно менять: Текст сообщений, дизайн кнопок.

📁 API (обработчики AJAX/JSON)
6. /api/auctions/create.php
Назначение: Обработка отправки формы создания аукциона.

Функционал:

Валидация всех полей.

Проверка лимита черновиков (2 неоплаченных).

Сохранение черновика в listings со статусом draft.

Загрузка фотографий в /uploads/auctions/.

Создание платежа через PaymentService.

Возврат JSON с URL для оплаты.

Что можно менять: Валидационные правила, путь для фото, платёжную логику.

7. /api/auctions/bid.php
Назначение: Обработка ставки (основное ядро).

Функционал:

Проверка авторизации, CSRF, блокировки, согласия с правилами.

Проверка суммы (текущая ставка + шаг).

Проверка наличия ставок (3 бесплатных на аукцион, платные глобально, макс 20 платных на аукцион).

Транзакция с FOR UPDATE.

Списание ставки (бесплатная / платная), обновление счётчиков.

Генерация анонимного ID и цвета.

Сохранение ставки в auction_bids.

Soft close (продление на 5 минут при ставке в последние 5 минут).

Публикация события в Redis для SSE.

Уведомление предыдущему лидеру.

Возврат JSON с новыми остатками.

Что можно менять: Лимиты ставок (бесплатные, платные, максимум на аукцион), шаг продления, текст ошибок.

8. /api/auctions/buy_now.php
Назначение: Мгновенная покупка по фиксированной цене.

Функционал:

Проверка, что цена указана, аукцион активен.

Проверка, что покупатель не продавец.

Завершение аукциона, создание сделки.

Публикация в Redis события auction_ended.

Уведомления продавцу и покупателю.

Что можно менять: Логику создания сделки, уведомления.

9. /api/auctions/consent.php
Назначение: Сохранение согласия с правилами.

Функционал: Берёт текущую версию из auction_consent_versions, сохраняет запись в user_auction_consents с IP и user‑agent.

Что можно менять: Ничего, это служебный файл.

10. /api/auctions/consent_text.php
Назначение: Возвращает текст текущей версии правил для модального окна.

Что можно менять: Ничего.

11. /api/auctions/relist.php
Назначение: Бесплатное перевыставление аукциона (если не было ставок).

Функционал: Проверяет статус closed_no_bids, что free_relist_used = 0. Устанавливает новую дату окончания, меняет статус на active, помечает free_relist_used = 1.

Что можно менять: Длительность перевыставления (по умолчанию 7 дней).

12. /api/auctions/accept_winner.php
Назначение: Продажа лота победителю при недостижении резервной цены.

Функционал: Продавец нажимает кнопку, создаётся сделка с ценой максимальной ставки.

Что можно менять: Ничего.

13. /api/auctions/delete.php
Назначение: Удаление черновика или завершённого аукциона (без продажи).

Функционал: Устанавливает deleted_at и status='deleted' для статусов draft, closed_no_bids, reserve_not_met, cancelled.

Что можно менять: Разрешённые для удаления статусы.

14. /api/auctions/accept_offer.php
Назначение: API принятия/отказа от предложения второму участнику.

Функционал: Принимает offer_id и action (accept/decline), обновляет second_offers, при принятии создаёт сделку, обновляет аукцион, уведомляет.

Что можно менять: Ничего.

15. /api/deals/confirm.php
Назначение: Подтверждение сделки продавцом или покупателем.

Функционал: Обновляет seller_confirmed или buyer_confirmed. Если оба подтвердили – статус confirmed, создаётся чат, отправляются уведомления.

Что можно менять: Время жизни сделки (сейчас 24 часа), создание чата.

16. /api/payments/buy_bids.php
Назначение: Создание платежа на покупку 10 ставок за 50₽ через ЮKassa.

Что можно менять: Цену и количество ставок (в методе createExtraBidsPayment).

📁 Служебные файлы
17. /sse/auction.php
Назначение: Server‑Sent Events endpoint для живых обновлений.

Функционал: Устанавливает text/event-stream заголовки, подключается к Redis, подписывается на канал auction:{id}, при получении сообщения отправляет event: update с данными, держит соединение открытым.

Что можно менять: Таймауты, обработку ошибок.

18. /services/RedisService.php
Назначение: Singleton для подключения к Redis.

Функционал: getInstance() возвращает объект Redis.

Что можно менять: Параметры подключения (хост, порт).

19. /services/PaymentService.php
Назначение: Работа с ЮKassa.

Методы:

createAuctionListingPayment($listingId, $amount) – платёж за размещение.

createExtraBidsPayment($userId, $count = 10) – платёж за 10 ставок.

Что можно менять: Суммы, метаданные.

20. /webhook/yookassa.php
Назначение: Обработка уведомлений от ЮKassa.

Функционал: При payment.succeeded проверяет metadata['type']. Для auction_listing – активирует аукцион (оплата размещения). Для extra_bids – увеличивает extra_bids_balance пользователя на 10.

Что можно менять: Ничего.

21. /includes/auction_functions.php
Назначение: Вспомогательные функции для аукционов.

Функции:

calculateCommission($price) – расчёт комиссии 2% (min 50, max 2000).

calculateListingFee($startBid) – плата за размещение (30–100 ₽).

formatTimeLeft($seconds) – форматирование оставшегося времени.

generateAnonymousId($listingId, $userId) – генерация анонимного ID (4 символа).

generateColor($anonId) – цвет по хешу ID.

getCurrentConsentText(), getCurrentConsentVersion(), hasActiveConsent($userId) – работа с правилами.

isUserBlocked($userId) – проверка блокировки.

checkRemainingBids($userId, $listingId) – проверка остатка ставок.

sendAuctionNotification($userId, $type, $data) – отправка уведомления через NotificationService.

blockUserFromAuctions($userId, $days, $reason) – блокировка.

getCurrentMaxBid($listingId), getBidsCount($listingId) – получение данных.

Что можно менять: Логику расчёта комиссии, лимиты ставок, длительность блокировки.

22. /payment/success.php
Назначение: Страница успешной оплаты после возврата из ЮKassa.

Функционал: Принимает параметры type (listing или extra_bids), отображает сообщение, через 5 секунд перенаправляет в личный кабинет или на страницу аукциона.

Что можно менять: Текст сообщений, время редиректа.

📁 Воркеры (cron / supervisor)
23. /workers/close_auctions.php
Запуск: Каждые 5 минут (cron).

Назначение: Завершение аукционов с истёкшим временем.

Функционал: Находит активные аукционы с auction_end_at <= NOW(). Если ставок нет – closed_no_bids. Если ставка есть и резерв достигнут – создаёт сделку awaiting_confirmation. Если резерв не достигнут – reserve_not_met и уведомление продавцу.

Что можно менять: Периодичность запуска, логику создания сделки.

24. /workers/handle_unpaid.php
Запуск: Каждые 5 минут (cron).

Назначение: Обработка неоплаченных сделок.

Функционал: Находит сделки с истёкшим сроком подтверждения. Если покупатель не подтвердил – блокирует на 30 дней. Если включено auto_offer_second – создаёт предложение второму участнику. Если продавец не подтвердил (а покупатель подтвердил) – переводит сделку в спор.

Что можно менять: Срок блокировки, срок предложения второму.

25. /workers/charge_commissions.php
Запуск: Раз в час (cron).

Назначение: Начисление комиссии после подтверждения сделки.

Функционал: Находит подтверждённые сделки без начисленной комиссии, рассчитывает комиссию 2% (min 50, max 2000), создаёт запись в seller_commissions (статус pending, срок 7 дней), уведомляет продавца.

Что можно менять: Периодичность, срок оплаты комиссии.

26. /workers/cleanup_expired.php
Запуск: Раз в сутки (cron).

Назначение: Очистка устаревших данных.

Функционал: Удаляет черновики старше 1 дня без оплаты, ставки старше 1 года, логи аудита старше 3 лет, истёкшие блокировки.

Что можно менять: Сроки хранения данных.

27. /workers/process_email_queue.php
Запуск: Каждую минуту (cron) или supervisor.

Назначение: Обработка очереди email-уведомлений.

Функционал: Вызывает processEmailQueue() из NotificationService, который отправляет до 100 писем за запуск, используя Redis или файловую очередь.

Что можно менять: Лимит писем за запуск.

📁 Административная панель
28. /admin/auctions.php
Назначение: Административный интерфейс для управления аукционами.

Функционал:

Фильтрация по статусу, типу, категории, городу, продавцу, дате.

Просмотр списка аукционов с действиями: принудительное завершение, отмена, удаление.

Блокировка/разблокировка пользователей.

Просмотр ставок по аукциону.

Все действия логируются в audit_log.

Что можно менять: Список разрешённых действий, фильтры, внешний вид.

📁 База данных (таблицы, создаваемые установщиком)
Установщик (install.php) создаёт все необходимые таблицы, включая:

users – доп. поля extra_bids_balance, phone_verified.

listings – аукционные поля (auction_type, start_bid, reserve_price, min_bid_increment, auction_end_at, hidden_bids, auto_offer_second, listing_fee_paid, winner_id, final_price, auction_status, enable_3d_effect, buy_now_price, free_relist_used).

auction_bids, auction_participants, deals, second_offers, reverse_offers, user_blocks, auction_consent_versions, user_auction_consents, audit_log, seller_commissions, auction_chats, chat_messages, notification_templates, notifications, notification_logs, user_notification_settings и др.

📌 Вывод
Все файлы перечислены, их функционал описан. Модуль полностью автономен и интегрируется с существующими сервисами (Database, NotificationService). Никаких «заглушек» нет, всё готово к работе. Если вы хотите изменить какие‑то параметры (например, количество бесплатных ставок, цену пакета, комиссию), это легко сделать в auction_functions.php и PaymentService.php.
