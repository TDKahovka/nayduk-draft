1. Фронт-контроллер и маршрутизация
Файл: public/index.php
Роль: единая точка входа.
Функции:

Обрабатывает API‑запросы (маршруты, начинающиеся с /api/) – подключает соответствующий файл.

Отдаёт статику (css, js, изображения) с заголовками кэширования.

Маршрутизирует обычные страницы (главная, категории, объявления, магазины, профиль, авторизация, бизнес‑кабинет, офферы).

Встраивает содержимое страницы в шаблон views/layouts/main.php (если страница не отправила свои заголовки).

Зависимости: vendor/autoload.php, includes/functions.php, Database, views/layouts/main.php.

2. Шаблоны и общие элементы
Файл: views/layouts/main.php
Роль: главный шаблон для всех страниц.
Функции:

Подключает CSS (Hugeicons, Toastify, темы).

Микроразметка Schema.org.

Подвал.

Скрипты для CSRF‑токена, темы, уведомлений, флеш‑сообщений.

Вывод содержимого страницы.

3. Общие API и вспомогательные файлы
Файл: includes/functions.php

Функции:

Генерация slug, транслитерация.

Управление сессией, флеш‑сообщениями.

Работа с базой (обёртки).

Загрузка/оптимизация изображений (ImageOptimizer).

Файл: includes/header.php и includes/footer.php
Роль: подключаются в старых страницах (если они не используют layout). Сейчас используются редко, но оставлены для совместимости.

Файл: services/Database.php
Роль: класс для работы с БД.
Функции:

Получение экземпляра (Singleton).

Выполнение запросов, подготовка, транзакции.

Методы fetchOne, fetchAll, fetchCount, insert, update, delete.

Кэширование результатов через Redis/файлы.

Получение пользователя по ID.

Файл: services/ImageOptimizer.php
Роль: оптимизация изображений (сжатие, изменение размера, конвертация в WebP).
Функции:

optimize($source, $options) – обрабатывает изображение, сохраняет в uploads/optimized/, возвращает путь.

Используется для аватаров, фото товаров, баннеров.

Файл: services/NotificationService.php
Роль: отправка уведомлений (email, push, SMS – заглушка).
Функции:

send($userId, $type, $data) – создаёт запись в notifications и отправляет email через очередь.

Файл: services/GeoService.php
Роль: работа с геоданными (город по IP, автодополнение).
Функции:

getUserCity($userId) – возвращает город из профиля, куки, IP.

suggestCities($query, $limit) – поиск городов в базе.

reverseGeocode($lat, $lng) – определение города по координатам (Nominatim).

8. Платежи через ЮKassa
Файл: api/payments/create.php
Роль: создание платежа в ЮKassa.
Функции:

Принимает type (subscription, storage, banner, order), amount, description, order_id.

Формирует запрос к API ЮKassa, возвращает confirmation_url.

Сохраняет запись в payment_orders.

Файл: api/payments/webhook.php
Роль: обработка уведомлений от ЮKassa.
Функции:

Проверяет событие payment.succeeded.

Обновляет payment_orders статус на paid.

В зависимости от типа:

order – меняет статус заказа на paid, уведомляет продавца и покупателя.

subscription – обновляет plan и plan_expires_at в shops, устанавливает is_business=1.

storage – увеличивает storage_limit_mb магазина.

banner – создаёт запись в business_banners со статусом pending.

13. Что можно улучшить / добавить
Документация – нужно описать для администратора, как настраивать ЮKassa, запускать воркеры, менять тарифы.

Чат – сейчас просто кнопка, ведущая на страницу /chat?user=.... Нужно реализовать полноценный мессенджер (WebSockets или Long Polling).

Уведомления в реальном времени – можно через Pusher или WebSockets, но уже есть SSE для аукционов (отдельно). Для сообщений и ставок тоже можно использовать SSE или общий канал.

Мобильное приложение – можно завернуть сайт в WebView или сделать PWA.

Дополнительные фильтры – на странице категории и поиска добавить фильтры по цене, состоянию, доставке.

Рекомендательная система – на главной и на странице товара показывать «с этим также покупают» на основе истории заказов.

Аналитика для администратора – дашборд с ключевыми метриками (новые пользователи, объявления, заказы, выручка).

Вебхуки для партнёров – уже заложено, но можно реализовать отправку уведомлений о новых заказах, изменении статусов.

Локализация – поддержка нескольких языков (можно начать с английского).

Тесты – юнит-тесты для критических API (ставки, платежи, создание объявлений).

Логирование ошибок – интеграция с Sentry или файловые логи с ротацией.

Оптимизация производительности – кэширование страниц магазинов, категорий, объявлений.

Интеграция с Яндекс.Маркет – импорт товаров, синхронизация остатков.

API для мобильных приложений – REST API с авторизацией по токену.

Аудит безопасности – регулярное сканирование на уязвимости, проверка CSRF, XSS.
